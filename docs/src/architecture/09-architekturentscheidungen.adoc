[[_entscheidungsprozess]]
=== Entscheidungsprozess

Architekturentscheidungen folgen diesem Prozess:

1. **Problem identifizieren** - Kontext und Constraints klären
2. **Alternativen evaluieren** - Verschiedene Lösungsansätze bewerten
3. **Entscheidung treffen** - Begründete Wahl mit Konsequenzen
4. **ADR erstellen** - Dokumentation in Markdown-Format
5. **Review** - Team-Review und Feedback
6. **Implementieren** - Technische Umsetzung
7. **Validieren** - Überprüfung der Entscheidung im Betrieb

[[_aenderung_von_entscheidungen]]
=== Änderung von Entscheidungen

ADRs sind **immutable**. Bei Änderungen:

1. Neues ADR erstellen mit Referenz auf altes ADR
2. Altes ADR auf Status "Superseded" setzen
3. Begründung für Änderung dokumentieren

Beispiel:
```
**Status:** Superseded by ADR-042
**Superseded Date:** 2026-03-15
```
